Introduo

A preparao dos dados uma etapa fundamental no ciclo de vida de um projeto de cincia de dados. O objetivo principal desta fase transformar dados brutos e heterogneos em um formato limpo e estruturado, pronto para ser utilizado na modelagem. Esta transformao essencial para garantir a acurcia e a eficincia dos modelos preditivos que sero desenvolvidos posteriormente. A preparao dos dados envolve vrias atividades, incluindo a limpeza, a transformao e a integrao dos dados, cada uma delas crucial para eliminar inconsistncias e preencher lacunas identificadas na fase de Entendimento dos Dados.

A importncia da preparao dos dados no pode ser subestimada, pois dados mal preparados podem levar a modelos imprecisos e concluses enganosas. A fase de Entendimento dos Dados forneceu insights valiosos sobre a qualidade e a estrutura dos dados, identificando problemas como dados faltantes, outliers e a necessidade de normalizao. Agora, com essas informaes, podemos proceder preparao dos dados de maneira sistemtica e eficaz. Esta fase conecta diretamente com as fases anteriores de Entendimento do Negcio e Entendimento dos Dados, assegurando que as aes tomadas sejam alinhadas com os objetivos do projeto e baseadas em uma compreenso profunda dos dados disponveis.

Reviso das Propostas de Alteraes

Durante a fase de Entendimento dos Dados, vrias propostas de alteraes foram sugeridas para melhorar a qualidade dos dados. Entre as principais propostas estavam:

  • Remoo de Outliers: Identificamos valores extremos que poderiam distorcer as anlises e os resultados dos modelos. A remoo desses outliers foi justificada pela anlise estatstica que mostrou que esses valores no representavam o comportamento tpico dos dados.
  • Imputao de Valores Ausentes: Detectamos que alguns registros tinham dados faltantes. A imputao desses valores crucial para manter a integridade do conjunto de dados. Mtodos como imputao pela mdia, mediana ou modas foram sugeridos com base na natureza das variveis afetadas.
  • Transformao de Variveis Categricas em Numricas: Variveis categricas foram identificadas como necessrias para transformao, uma vez que a maioria dos algoritmos de aprendizado de mquina requer variveis numricas. Tcnicas como one-hot encoding foram recomendadas para essa transformao.
  • Normalizao e Padronizao de Dados: Algumas variveis precisavam ser normalizadas ou padronizadas para garantir que todas estivessem na mesma escala, facilitando o processo de modelagem e aumentando a preciso dos modelos.

Essas propostas foram fundamentadas na anlise dos dados realizada anteriormente e visavam eliminar inconsistncias, melhorar a qualidade dos dados e prepar-los para a modelagem preditiva.

Remoo de Outliers, Normalizao e Padronizao de Dados

Nesta sesso, abordamos a remoo de outliers, a normalizao e a padronizao dos dados para garantir a qualidade e a consistncia das anlises subsequentes. As etapas especficas realizadas incluem:

  1. Remoo de Outliers: Para mitigar o impacto dos valores extremos nos dados, aplicamos a tcnica de winsorizao, truncando os valores mais baixos e mais altos com base em percentis especficos. Isso ajudou a reduzir a influncia dos outliers sem eliminar completamente os dados.
  2. Normalizao: Aps a remoo dos outliers, normalizamos as variveis para o intervalo [0, 1]. Este processo rescalou os dados, garantindo que todas as variveis numricas tivessem a mesma escala, facilitando comparaes diretas entre elas.
  3. Padronizao: Alm da normalizao, aplicamos a padronizao para variveis com diferentes unidades de medida. A padronizao transformou os dados de forma que cada varivel tivesse uma mdia de 0 e um desvio padro de 1, eliminando as discrepncias causadas por diferentes escalas e unidades.

Idade

A transformao aplicada varivel age consistiu em duas etapas principais. Primeiramente, para reduzir a assimetria positiva e lidar com outliers de idades extremas, foi utilizada a funo logartmica da varivel age elevada a $\frac{1}{2.15}$. Esta transformao ajudou a mitigar a influncia dos valores extremos e a melhorar a simetria da distribuio.

Na etapa seguinte, a varivel transformada foi submetida a uma winsorizao com limites especficos para truncar os valores extremos. Os limites utilizados foram [0.003, 0.001], significando que os 0,3% dos menores valores foram substitudos pelo valor no percentil de 0,3%, e os 0,1% dos maiores valores foram substitudos pelo valor no percentil de 99,9%. Esse procedimento ajudou a reduzir o impacto dos outliers extremos na anlise.

Aps a winsorizao, a varivel foi normalizada para o intervalo [0, 1], a fim de padronizar a escala com as outras variveis numricas e facilitar a comparao e anlise subsequente.

In[]:
Xt['age'] = mstats.winsorize(np.log(X['age'])**(1/2.15), limits=[0.003, 0.001])
Xt['age'] = (Xt['age'] - Xt['age'].min()) / (Xt['age'].max() - Xt['age'].min())

Equilibrio

Para aprimorar a qualidade dos dados da varivel balance, adotou-se uma abordagem em duas etapas. Primeiramente, foi aplicada a transformao do mdulo seguida da raiz stima. Especificamente, o valor absoluto da varivel balance foi elevado a $\frac{1}{7}$ (ou seja, a raiz stima). Essa transformao ajudou a reduzir significativamente a assimetria dos dados, embora no tenha eliminado completamente os outliers ou ajustado todas as observaes com valor zero.

Em seguida, foi realizada a winsorizao dos dados transformados com limites [0.1, 0.03]. Isso significa que os 10% dos valores mais baixos foram substitudos pelo valor no percentil de 10%, e os 3% dos valores mais altos foram substitudos pelo valor no percentil de 97%. Essa etapa visou minimizar o impacto dos outliers extremos.

Por fim, a varivel balance foi normalizada para o intervalo [0, 1] para padronizar a escala com as demais variveis numricas e facilitar anlises futuras.
